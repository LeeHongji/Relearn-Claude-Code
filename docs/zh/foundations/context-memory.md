# 上下文与记忆

所有 coding agent 最终都会遇到同一个问题：模型上下文窗口是有限的。

Claude Code 把上下文管理当成核心系统来做。

## 上下文不只是聊天记录

它还包括：

- 系统提示词；
- git 状态；
- `CLAUDE.md` 之类的项目说明；
- 记忆文件；
- 最近工具结果；
- 会话动态状态。

`src/context.ts` 是最重要的入口之一。

## 记忆系统

`src/memdir/` 展示了一个很实用的工程思路：先从**可读、可控、可截断的文件式记忆**开始，而不是一上来就假设一个神奇的黑盒数据库。

## 压缩系统

关键文件：

- `services/compact/compact.ts`
- `services/compact/autoCompact.ts`

当上下文接近上限时，运行时必须主动压缩，否则会出现：

- 忘记刚改过的文件；
- 历史越来越贵；
- 会话中断；
- 错误恢复困难。

## 一个教学结论

上下文工程不是“多塞一点信息进去”，而是“在有限窗口里放最有用、最可恢复的信息”。
