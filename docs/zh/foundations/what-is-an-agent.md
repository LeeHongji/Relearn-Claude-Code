# 什么是 AI Agent？

AI agent 不是“一个大模型”这么简单。它至少由下面几部分组成：

- **状态**：消息历史、文件状态、任务状态、用户偏好；
- **工具**：读文件、改代码、执行命令、调用 API；
- **策略**：什么时候继续、什么时候重试、什么时候请求权限；
- **记忆**：跨轮次、跨会话保留什么信息；
- **控制流**：如何把“思考—行动—观察”串起来。

## 最简单的理解方式

1. 读取当前任务和环境；
2. 问模型下一步要做什么；
3. 如果只是文本，就输出；
4. 如果是工具调用，就执行工具；
5. 把工具结果再喂回模型；
6. 重复直到任务完成。

这就是 coding agent 的核心。

## 为什么 coding agent 比聊天机器人难得多？

因为它面对的是：

- 大代码库；
- 真正有副作用的 shell 命令；
- 长时间运行任务；
- 中途失败和恢复；
- 模糊需求；
- 持续变化的仓库状态。

所以真正的 agent 一定会长出很多“非模型”子系统。

## 一个重要结论

> 好的 agent 往往首先是一个好的系统设计，而不是一个更长的 prompt。
